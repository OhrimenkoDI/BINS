# Напоминание Codex: работаем по-взрослому

Если задача уже сформулирована в векторах, матрицах или кватернионах — не сводить её без необходимости к углам Эйлера.

## Управление ориентацией

- Три компоненты угловой скорости `(wx, wy, wz)` — это один вектор, а не три последовательных поворота.
- За интервал `dt` сначала вычислить единый вектор углового перемещения:

  `dTheta = omega * dt`.

- Его длина — единый угол поворота:

  `Angle = |dTheta|`.

- Его нормализованное направление — единая ось поворота:

  `Axis = dTheta / Angle`.

- Создать один кватернион приращения:

  `DeltaQ = Quaternion(Axis, Angle)`.

- Применить его одним умножением и нормализовать результат:

  `Q := Normalize(Q * DeltaQ)` — локальная угловая скорость;

  `Q := Normalize(DeltaQ * Q)` — глобальная угловая скорость.

## Перед реализацией

1. Проверить физический смысл входных данных и систему координат.
2. Найти уже работающий алгоритм в исходном проекте и сохранить его математику, каналы и знаки осей.
3. Не подменять одновременное векторное вращение последовательностью `Yaw/Pitch/Roll`.
4. Помнить, что умножение кватернионов некоммутативно.
5. Использовать половинный угол только внутри формулы компонентов кватерниона; не делить угол повторно, если это уже делает конструктор.
6. После изменения обязательно собрать проект и проверить поведение в покое: при нулевом входе ориентация не должна изменяться.

Главное правило: сначала понять геометрию и физику, затем писать код.
