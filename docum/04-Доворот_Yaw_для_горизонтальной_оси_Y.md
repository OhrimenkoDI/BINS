# Доворот Yaw для горизонтального положения оси Y

## 1. Постановка задачи

Используются правые системы координат:

- Body FRD: X вперёд, Y вправо, Z вниз;
- NED: X север, Y восток, Z вниз;
- `q = q_nb` переводит Body → NED;
- положительный Yaw выполняется вокруг локальной оси Body Z;
- локальное приращение применяется справа: `qnew = q ⊗ DeltaQz`.

Платформа наклонена. Требуется, используя только Yaw, сделать локальную ось Y горизонтальной.

Yaw не изменяет наклон оси Z и не может выровнять всю платформу. Он может совместить Y с линией пересечения наклонной плоскости платформы и горизонтальной плоскости.

> **Обозначение:** угол этого специального доворота обозначается `ψ_align`. Это не стандартный Euler Yaw `ψ`, описывающий текущую ориентацию аппарата в документе 05. `ψ_align` является управляющим приращением вокруг локальной Z, вычисленным для отдельной задачи выравнивания оси Y.

## 2. Вертикальные компоненты локальных осей

Оси платформы, выраженные в NED:

`Xn = q ⊗ Xb ⊗ q⁻¹`,

`Yn = q ⊗ Yb ⊗ q⁻¹`.

Их вертикальные компоненты:

`XD = Xn.Z`,

`YD = Yn.Z`.

В NED ось считается горизонтальной, когда её компонента Z равна нулю.

## 3. Поворот осей при локальном Yaw

После доворота на угол `ψ_align`:

`X'n = cos(ψ_align)·Xn + sin(ψ_align)·Yn`,

`Y'n = −sin(ψ_align)·Xn + cos(ψ_align)·Yn`.

Вертикальная компонента новой оси Y:

`Y'D = −XD·sin(ψ_align) + YD·cos(ψ_align)`.

Условие горизонтальности:

`Y'D = 0`.

Отсюда:

`ψ_align = atan2(YD, XD)`.

Для текущей модели:

```text
XD = oX3.z
YD = oY3.z
YawAlign = atan2(YD, XD)
```

Кватернион доворота:

`DeltaQz = Quaternion((0,0,1), ψ_align)`.

Новая ориентация:

`qnew = q ⊗ DeltaQz`.

## 4. Два решения и минимальный доворот

Если `ψ_align` является решением, то `ψ_align + π` также является решением. Во втором случае горизонтальная ось Y направлена противоположно.

Если конкретное направление `+Y` не задано, выбирается минимальный по модулю угол. Его можно привести к диапазону `−π/2…+π/2`:

```text
ψ_align = atan2(YD, XD)
если ψ_align >  π/2: ψ_align = ψ_align − π
если ψ_align < −π/2: ψ_align = ψ_align + π
```

Если ось Y должна смотреть в заданную горизонтальную сторону, например на восток, решение выбирается дополнительной проверкой скалярного произведения новой Y с требуемым направлением.

## 5. Горизонтальная платформа

Если платформа уже горизонтальна:

`XD = 0`, `YD = 0`.

Тогда `atan2(0,0)` неопределён, что соответствует физике: ось Y горизонтальна при любом Yaw.

Необходима проверка:

`sqrt(XD² + YD²) < ε`.

В этом случае принимается `ψ_align = 0`.

## 6. Расчёт непосредственно из кватерниона

Пусть нормализованный кватернион:

`q = (w,x,y,z)`.

При преобразовании `vn = q ⊗ vb ⊗ q⁻¹` вертикальные компоненты локальных осей равны:

`XD = 2(xz − wy)`,

`YD = 2(yz + wx)`.

Поэтому:

`ψ_align = atan2(2(yz + wx), 2(xz − wy))`.

Общий множитель 2 можно убрать:

`ψ_align = atan2(yz + wx, xz − wy)`.

Здесь `w,x,y,z` — компоненты кватерниона.

## 7. Проверка направления команды

Формула соответствует соглашению:

`qnew = q ⊗ DeltaQz`.

Если исполнительная система считает положительный Yaw в обратном направлении, команда должна иметь знак `−ψ_align`. Знак проверяется экспериментом: наклонить платформу вокруг X. При `YD ≠ 0` и `XD ≈ 0` требуемый доворот будет близок к `±90°`.

## 8. Итог

Основная формула:

`ψ_align = atan2(Yn.Z, Xn.Z)`.

Или непосредственно из кватерниона:

`ψ_align = atan2(yz + wx, xz − wy)`.

После вычисления выбирается ближайшее из решений `ψ_align` и `ψ_align ± π`, а для почти горизонтальной платформы возвращается нулевой доворот.
