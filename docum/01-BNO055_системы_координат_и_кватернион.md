# BNO055: системы координат и кватернион модели летательного аппарата

## 1. Принятые системы координат

Для математической модели используется классическое авиационное соглашение.

### Связанная система аппарата Body FRD

- `Xb` — вперёд, вдоль продольной оси аппарата.
- `Yb` — вправо, в сторону правого крыла.
- `Zb` — вниз.

Начало координат желательно располагать в центре масс. Это правая система: `Xb × Yb = Zb`.

Вектор угловой скорости в связанных осях:

`ωb = (p, q, r)`,

где `p` — скорость крена вокруг Xb, `q` — скорость тангажа вокруг Yb, `r` — скорость рыскания вокруг Zb.

### Навигационная система NED

- `Xn` — север.
- `Yn` — восток.
- `Zn` — вниз.

Это также правая система: `Xn × Yn = Zn`. При горизонтальном положении аппарата и направлении носа на север оси Body и NED совпадают, поэтому единичный кватернион означает исходную ориентацию.

## 2. Смысл кватерниона

Кватернион `q_nb` переводит вектор из связанных координат Body в навигационные координаты NED:

`vn = q_nb ⊗ vb ⊗ q_nb*`.

В коде:

```pascal
VectorN := Q * VectorB * Q.Inverse;
```

Направления локальных осей аппарата в NED:

```pascal
OXn := Q * OXb * Q.Inverse;
OYn := Q * OYb * Q.Inverse;
OZn := Q * OZb * Q.Inverse;
```

## 3. Интегрирование угловой скорости

Компоненты гироскопа являются одним вектором угловой скорости, а не тремя углами Эйлера.

За интервал `dt`:

`dTheta = ωb · dt`.

Единый угол и единая ось поворота:

`Angle = |dTheta|`,

`Axis = dTheta / Angle`.

Создаётся один кватернион приращения:

`DeltaQ = Quaternion(Axis, Angle)`.

Поскольку угловая скорость задана в локальных осях аппарата:

```pascal
DX := P * Dt;
DY := QRate * Dt;
DZ := R * Dt;

Angle := Sqrt(DX*DX + DY*DY + DZ*DZ);
if Angle > 0 then
begin
  Axis := TPoint3D.Create(DX / Angle, DY / Angle, DZ / Angle);
  DeltaQ := TQuaternion.Create(Axis, Angle);
  Orientation := (Orientation * DeltaQ).Normalize;
end;
```

Эквивалентное дифференциальное уравнение:

`q̇_nb = 1/2 · q_nb ⊗ [0, p, q, r]`.

Углы Эйлера для интегрирования ориентации не используются.

## 4. Установка и оси BNO055

После настройки логические оси BNO055 должны совпадать с Body FRD:

- `BNO X` → нос аппарата.
- `BNO Y` → правое крыло.
- `BNO Z` → вниз.

BNO055 позволяет переставлять оси и менять их знаки регистрами:

- `AXIS_MAP_CONFIG`, адрес `0x41`.
- `AXIS_MAP_SIGN`, адрес `0x42`.

Настройка выполняется в `CONFIGMODE` и должна повторяться после каждого включения питания. Регистры обеспечивают только перестановку осей и изменение знаков. Для установки датчика под произвольным углом нужен постоянный установочный кватернион.

Официальная документация: [Bosch BNO055 Datasheet](https://www.bosch-sensortec.com/media/boschsensortec/downloads/datasheets/bst-bno055-ds000.pdf), раздел 3.4.

## 5. Установочный кватернион

Обозначения:

- `S` — физические оси датчика.
- `B` — связанные оси аппарата FRD.
- `N` — навигационные оси NED.
- `q_ns` — ориентация датчика, полученная от BNO055.
- `q_bs` — постоянный поворот Sensor → Body.

Так как `q_sb = q_bs⁻¹`, ориентация аппарата вычисляется как:

`q_nb = q_ns ⊗ q_sb`.

В коде:

```pascal
QBody := QBNO * QSensorToBody.Inverse;
```

Если axis remap уже совместил логические оси BNO055 с FRD:

```pascal
QSensorToBody := TQuaternion.Identity;
QBody := QBNO;
```

## 6. Формат кватерниона BNO055

BNO055 выдаёт компоненты в порядке `W, X, Y, Z`. Каждая компонента является знаковым 16-битным числом. Масштаб: одна единица кватерниона равна `2^14` LSB.

```pascal
QBNO.w := RawW / 16384.0;
QBNO.x := RawX / 16384.0;
QBNO.y := RawY / 16384.0;
QBNO.z := RawZ / 16384.0;
QBNO := QBNO.Normalize;
```

Регистры кватерниона находятся по адресам `0x20…0x27`.

## 7. Формат Windows или Android

Бит 7 регистра `UNIT_SEL` выбирает формат ориентации:

- `0` — Windows.
- `1` — Android.

Формат необходимо задавать явно при каждой инициализации и не менять во время работы. Физическая модель не должна зависеть от названий формата: вход BNO055 один раз приводится к принятому соглашению FRD/NED.

## 8. Проверка направления кватерниона

Направление преобразования нельзя оставлять на предположении. Оно проверяется экспериментально.

1. Аппарат горизонтален, нос направлен на север: ожидается `Q ≈ (1, 0, 0, 0)` после начальной привязки.
2. Положительный крен вокруг Xb должен формировать положительное вращение вокруг X.
3. Аналогично проверяются положительные повороты вокруг Yb и Zb.

Для относительной ориентации от момента включения:

`q_relative = q0⁻¹ ⊗ q_BNO`,

где `q0` — показание BNO055 в исходном положении.

Если все вращения получаются обратными, вероятно, датчик выдаёт обратное направление преобразования относительно принятого `q_nb`. Тогда используется:

```pascal
QBody := QBNO.Inverse;
```

Ошибку только одной оси следует исправлять через axis remap/sign, а не обращением всего кватерниона.

## 9. Поступательная динамика

Основное уравнение в NED:

`v̇n = R(q_nb) · fb + gn`,

где `fb` — удельная сила в Body, а вектор гравитации:

`gn = (0, 0, +9.80665) м/с²`.

Положительный знак обусловлен тем, что ось Zn направлена вниз.

Рекомендуемый состав состояния модели:

```text
PositionNED       = (North, East, Down)
VelocityNED       = (Vn, Ve, Vd)
Orientation       = q_nb
AngularRateBody   = (p, q, r)
SpecificForceBody = (fx, fy, fz)
```

## 10. Итоговое соглашение

```text
Корпус:     FRD — X вперёд, Y вправо, Z вниз
Навигация:  NED — X север, Y восток, Z вниз
Кватернион: q_nb, преобразование Body → NED
Гироскоп:   (p, q, r) в Body
Обновление: q := Normalize(q * DeltaQ)
BNO055:     axis remap в FRD, компоненты W/X/Y/Z делятся на 16384
```
