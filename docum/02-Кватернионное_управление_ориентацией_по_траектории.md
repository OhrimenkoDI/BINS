# Кватернионное управление ориентацией по траектории

## 1. Постановка задачи

Летательный аппарат находится на начальном уровне `H0` с произвольной ориентацией `q0`. На уровне `Hh` требуется получить ориентацию `qh`.

Управляющими заданиями являются компоненты угловой скорости в связанных осях Body FRD:

`ωcmd = (p_cmd, q_cmd, r_cmd)`,

где `p` — крен вокруг Body X, `q` — тангаж вокруг Body Y, `r` — рыскание вокруг Body Z.

Рекомендуемая структура:

```text
высота/время → заданный кватернион qref
                         ↓
               ошибка ориентации qe
                         ↓
             требуемая угловая скорость
                    (p, q, r)
                         ↓
             внутренний контур гироскопов
```

## 2. Slerp как генератор ориентационной траектории

Для параметра прогресса `s ∈ [0,1]`:

`qref(s) = Slerp(q0, qh, s)`.

Slerp движется по кратчайшей геодезической и представляет единый результирующий поворот, а не последовательность углов Эйлера.

Кватернионы `q` и `−q` задают одинаковую ориентацию. Чтобы выбрать короткую дугу, перед Slerp необходимо обеспечить:

`dot(q0, qh) ≥ 0`.

Если скалярное произведение отрицательное, следует заменить `qh := −qh`.

## 3. Плавный параметр прогресса

При жёсткой привязке к высоте:

`s = clamp((H − H0)/(Hh − H0), 0, 1)`.

Линейный параметр создаёт скачки заданной угловой скорости в начале и конце. Для плавного старта и остановки используется S-кривая:

`τ(s) = 10s³ − 15s⁴ + 6s⁵`,

после чего:

`qref = Slerp(q0, qh, τ)`.

У этой функции нулевые скорость и ускорение на границах. Высоту следует фильтровать. Если вертикальное движение может останавливаться или менять направление, лучше использовать монотонный параметр прогресса либо временную траекторию с коррекцией по высоте.

## 4. Кватернион ошибки

Принято соглашение `q = q_nb`: кватернион переводит Body FRD → NED. Угловая скорость задаётся в локальных осях Body.

Локальная ошибка ориентации:

`qe = q⁻¹ ⊗ qref`.

Она удовлетворяет равенству:

`qref = q ⊗ qe`.

Чтобы использовать поворот не более 180°:

`qe.w < 0  ⇒  qe := −qe`.

## 5. Из ошибки в единый вектор вращения

Пусть:

`qe = (w, v)`, где `v = (x,y,z)`.

Тогда:

`θ = 2·atan2(|v|, w)`,

`e = v/|v|`,

`φ = θ·e`.

Вектор `φ` является логарифмическим отображением ошибки ориентации. Его компоненты выражены в связанных осях:

`φ = (φx, φy, φz)`.

Для малых ошибок допустимо приближение `φ ≈ 2v`, но для произвольной ориентации используется полная формула через `atan2`.

## 6. Задание угловой скорости

Базовый закон управления:

`ωcmd = KR · φ`.

То есть:

```text
p_cmd = Kx · φx
q_cmd = Ky · φy
r_cmd = Kz · φz
```

Это компоненты одного вектора угловой скорости, а не производные углов Эйлера.

Для слежения за движущимся заданием применяется упреждение:

`ωcmd = ωff + KR·φ`,

где `ωff` — угловая скорость самой заданной траектории, а `KR·φ` устраняет ошибку слежения.

## 7. Ограничение скорости

Если допустима одинаковая скорость вращения по всем направлениям, ограничивается норма целого вектора:

`|ωcmd| ≤ ωmax`.

При превышении:

`ωcmd := ωcmd · ωmax/|ωcmd|`.

Так сохраняется рассчитанная ось вращения. Независимое обрезание `p`, `q`, `r` изменяет ось.

При разных физических пределах приводов применяется эллипсоидальное ограничение:

`(p/pmax)² + (q/qmax)² + (r/rmax)² ≤ 1`.

## 8. Внутренний контур угловой скорости

Внешний кватернионный регулятор формирует `(p_cmd,q_cmd,r_cmd)`. Внутренний контур сравнивает их с гироскопом:

`τcmd = Kω · (ωcmd − ωmeas)`.

В практическом регуляторе также учитываются интегральная составляющая, моменты инерции, гироскопические связи, ограничения приводов и anti-windup.

## 9. Связь ориентации и траектории полёта

Крен и тангаж меняют направление тяги, поэтому ориентацию нельзя назначать независимо от поступательного движения. Внешний навигационный контур должен проверять физическую достижимость `qref` и наличие достаточной вертикальной составляющей тяги.

```text
Навигация: высота, скорость, положение
                    ↓
         допустимый qref
                    ↓
      регулятор ориентации SO(3)
                    ↓
             p_cmd/q_cmd/r_cmd
                    ↓
      регулятор угловой скорости
                    ↓
                 приводы
```

## 10. Итоговый закон

`qref = Slerp(q0, qh, τ)`,

`qe = q⁻¹ ⊗ qref`,

`ωcmd = ωff + KR·Log(qe)`.

Slerp определяет желаемую ориентационную траекторию. `Log(qe)` определяет единую локальную ось и угол ошибки. Регулятор преобразует их в единый вектор заданной угловой скорости `(p,q,r)`.
