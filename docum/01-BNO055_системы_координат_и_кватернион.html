BNO055: системы координат и кватернион модели летательного аппарата

1. Принятые системы координат

Связанная система аппарата Body FRD

  • Xb — вперёд.
  • Yb — вправо.
  • Zb — вниз.

Начало координат желательно располагать в центре масс. Это правая система: Xb × Yb = Zb. Вектор угловой скорости ωb = (p,q,r): крен вокруг Xb, тангаж вокруг Yb, рыскание вокруг Zb.

Навигационная система NED

  • Xn — север.
  • Yn — восток.
  • Zn — вниз.

При горизонтальном положении и направлении носа на север Body и NED совпадают, поэтому единичный кватернион задаёт исходную ориентацию.

2. Смысл кватерниона

q_nb переводит вектор из Body в NED:

vn = qnb ⊗ vb ⊗ qnb*
VectorN := Q * VectorB * Q.Inverse;
OXn := Q * OXb * Q.Inverse;
OYn := Q * OYb * Q.Inverse;
OZn := Q * OZb * Q.Inverse;

3. Интегрирование угловой скорости

Компоненты гироскопа образуют один вектор, а не три угла Эйлера. За шаг: dTheta = ωb·dt, Angle = |dTheta|, Axis = dTheta/Angle.

DX := P * Dt;
DY := QRate * Dt;
DZ := R * Dt;
Angle := Sqrt(DX*DX + DY*DY + DZ*DZ);
if Angle > 0 then
begin
  Axis := TPoint3D.Create(DX / Angle, DY / Angle, DZ / Angle);
  DeltaQ := TQuaternion.Create(Axis, Angle);
  Orientation := (Orientation * DeltaQ).Normalize;
end;
q̇nb = ½ qnb ⊗ [0,p,q,r]

4. Установка и оси BNO055

После настройки: BNO X → нос, BNO Y → правое крыло, BNO Z → вниз.

  • AXIS_MAP_CONFIG = 0x41
  • AXIS_MAP_SIGN = 0x42

Настройка выполняется в CONFIGMODE после каждого включения. Она поддерживает перестановку осей и знаки; произвольный угол установки требует постоянного установочного кватерниона. См. Bosch BNO055 Datasheet, раздел 3.4.

5. Установочный кватернион

S — Sensor, B — Body, N — NED. Если q_ns получен от датчика, а q_bs переводит Sensor → Body:

qnb = qns ⊗ qsb, где qsb = qbs−1
QBody := QBNO * QSensorToBody.Inverse;

Если axis remap уже совместил BNO055 с FRD, установочный кватернион единичный и QBody := QBNO.

6. Формат кватерниона BNO055

Порядок компонент: W, X, Y, Z. Это знаковые 16-битные значения, масштаб 214 LSB на единицу. Регистры: 0x20…0x27.

QBNO.w := RawW / 16384.0;
QBNO.x := RawX / 16384.0;
QBNO.y := RawY / 16384.0;
QBNO.z := RawZ / 16384.0;
QBNO := QBNO.Normalize;

7. Windows или Android

Бит 7 UNIT_SEL: 0 — Windows, 1 — Android. Формат задаётся явно при каждой инициализации. Вход датчика один раз приводится к принятому соглашению FRD/NED.

8. Проверка направления

  1. Горизонтально, нос на север: после привязки Q ≈ (1,0,0,0).
  2. Положительный крен проверяет ось X.
  3. Положительные тангаж и рыскание проверяют Y и Z.

Относительная ориентация: q_relative = q0⁻¹ ⊗ q_BNO. Если все направления обратны, проверить необходимость QBNO.Inverse. Ошибку одной оси исправлять через axis remap/sign.

9. Поступательная динамика

v̇n = R(qnb)·fb + gn,   gn=(0,0,+9.80665) м/с²
PositionNED       = (North, East, Down)
VelocityNED       = (Vn, Ve, Vd)
Orientation       = q_nb
AngularRateBody   = (p, q, r)
SpecificForceBody = (fx, fy, fz)

10. Итоговое соглашение

Корпус:     FRD — X вперёд, Y вправо, Z вниз
Навигация:  NED — X север, Y восток, Z вниз
Кватернион: q_nb, Body → NED
Гироскоп:   (p, q, r) в Body
Обновление: q := Normalize(q * DeltaQ)
BNO055:     axis remap в FRD, W/X/Y/Z делятся на 16384
