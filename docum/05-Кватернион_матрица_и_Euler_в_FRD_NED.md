# Кватернион, матрица поворота и Euler в Body FRD / NED

## 1. Единое соглашение

### Body FRD

- `Xb` — Forward, вперёд;
- `Yb` — Right, вправо;
- `Zb` — Down, вниз.

Это правая система: `Xb × Yb = Zb`. Вектор угловой скорости `ωb=(p,q,r)` содержит скорость крена вокруг X, тангажа вокруг Y и рыскания вокруг Z.

### Navigation NED

- `Xn` — North, север;
- `Yn` — East, восток;
- `Zn` — Down, вниз.

NED также является правой системой. При горизонтальном положении и направлении носа на север Body и NED совпадают.

## 2. Кватернион Body → NED

Используется кватернион Гамильтона `q_nb=(w,x,y,z)`. Он активно поворачивает вектор из Body в NED:

`vn = q_nb ⊗ vb ⊗ q_nb*`.

```pascal
VectorN := Q * VectorB * Q.Inverse;
```

Для единичного кватерниона `q⁻¹=q*=(w,−x,−y,−z)`.

## 3. Матрица `R_nb`

Для `q=(w,x,y,z)`:

```text
       | 1−2(y²+z²)   2(xy−wz)      2(xz+wy)    |
R_nb = | 2(xy+wz)     1−2(x²+z²)    2(yz−wx)    |
       | 2(xz−wy)     2(yz+wx)      1−2(x²+y²)  |
```

Преобразование:

`vn = R_nb · vb`.

Столбцы `R_nb` — оси Body, выраженные в NED:

| Столбец | Ось Body | Компоненты в NED |
|---|---|---|
| 1 | Forward, Xb | `(R11,R21,R31)` |
| 2 | Right, Yb | `(R12,R22,R32)` |
| 3 | Down, Zb | `(R13,R23,R33)` |

Обратное преобразование:

`R_bn = R_nb⁻¹ = R_nbᵀ`.

## 4. Проекции

| Вид | Экранные направления | Компоненты NED |
|---|---|---|
| сверху X–Y | North вверх, East вправо | N, E |
| сбоку X–Z | North вправо, Down вниз | N, D |
| спереди Y–Z | East вправо, Down вниз | E, D |

Каждая локальная ось берётся из соответствующего столбца `R_nb` и проецируется на нужные две координаты.

## 5. Интегрирование гироскопа

Гироскоп выдаёт один вектор:

`ωb=(p,q,r)`.

За шаг `dt`:

`dTheta = ωb·dt`,

`Angle = |dTheta|`,

`Axis = dTheta/Angle`.

Создаётся один кватернион приращения:

`DeltaQ = Quaternion(Axis,Angle)`.

Поскольку скорость выражена в локальных осях:

`q_nb := Normalize(q_nb ⊗ DeltaQ)`.

Дифференциальная форма:

`q̇_nb = 1/2·q_nb⊗[0,p,q,r]`.

Компоненты `(p,q,r)` не являются тремя углами Эйлера и не должны интегрироваться последовательными поворотами.

## 6. Euler ZYX только для отображения

При соглашении:

`R_nb = Rz(ψ)·Ry(θ)·Rx(φ)`

получаем:

`Pitch θ = −asin(R31)`,

`Roll φ = atan2(R32,R33)`,

`Yaw ψ = atan2(R21,R11)`.

Через кватернион:

```text
Roll  = atan2(2(wx+yz), 1−2(x²+y²))
Pitch = asin(2(wy−zx))
Yaw   = atan2(2(wz+xy), 1−2(y²+z²))
```

В FRD/NED:

- положительный Roll — правое крыло вниз;
- положительный Pitch — нос вверх;
- положительный Yaw вокруг Z вниз — поворот вправо.

При `|Pitch|=90°` возникает gimbal lock: Roll и Yaw отдельно не определены, но кватернион и матрица остаются корректными.

## 7. Однородная матрица

Для положения `rn=(N,E,D)`:

```text
       | R11 R12 R13 N |
T_nb = | R21 R22 R23 E |
       | R31 R32 R33 D |
       |  0   0   0  1 |
```

## 8. Согласование с BNO055

- После axis remap: BNO X к носу, BNO Y к правому крылу, BNO Z вниз.
- Компоненты читаются в порядке W, X, Y, Z.
- Знаковые 16-битные значения делятся на `2^14=16384`.
- Регистры кватерниона: `0x20…0x27`.
- Axis remap: `AXIS_MAP_CONFIG=0x41`, `AXIS_MAP_SIGN=0x42`.
- Формат Windows/Android задаётся явно и один раз приводится к FRD/NED.
- При произвольном угле монтажа применяется постоянный установочный кватернион.

## 9. Контрольные проверки

1. Горизонтально, нос на север: `q≈(1,0,0,0)`, `R≈I`.
2. Положительный Roll: правое крыло вниз, `R32>0`.
3. Положительный Pitch: нос вверх, `R31<0`.
4. Положительный Yaw: нос от севера к востоку, `R21>0`.
5. Для корректной матрицы: `RᵀR=I`, `det(R)=+1`.

## 10. Итог

```text
Body:       FRD — X вперёд, Y вправо, Z вниз
Navigation: NED — X север, Y восток, Z вниз
Quaternion: q_nb, Body → NED, W/X/Y/Z
Vector:     vn = q_nb × vb × inverse(q_nb)
Matrix:     vn = R_nb × vb
Gyroscope:  ωb = (p,q,r) в Body
Update:     q_nb := Normalize(q_nb × DeltaQ)
Euler:      только отображение ZYX
```
